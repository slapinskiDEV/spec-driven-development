# Engineering Principles

These are general engineering principles that should be followed in this project and any other projects using the `.agent` guidance.

## Core Values

- **Simplicity:** Prefer simple, clear solutions over clever, complex ones. Code should be easy to understand at a glance.
- **Maintainability:** Write code with the future maintainer in mind. Aim for code that is easy to change and extend.
- **Readability:** Prioritize human readability of the codebase. Naming should be clear and descriptive.
- **Modularity:** Design the system as a collection of small, focused, and loosely coupled components.
- **Consistency:** Follow established patterns and styles throughout the codebase to reduce cognitive load.
- **Traceability:** Ensure that every major decision and activity is recorded and understandable in the project history.

## General Guidelines

- **Review First:** Major changes should be planned and reviewed before implementation begins.
- **Automated Testing:** Aim for high-quality automated tests that provide confidence in the system's correctness.
- **Continuous Improvement:** Regularly refactor and improve the codebase as part of the normal development process.
- **Pragmatism:** Choose the right tool for the job. Avoid over-engineering and strictly dogmatic approaches.
- **Documentation:** Keep documentation (like this `.agent` workspace) and code documentation (using `/** ... */` for functions) accurate and up to date.
